Spacing

The spacing system manages margins and gaps through a 3-tier token system in the LINE GIFTSHOP platform. It ensures consistent spatial relationships between elements across all components.

Semantic Component

3-Tier Token System

Systematic Spacing Structure

Spacing tokens are organized in three tiers: Dimension (primitive values), Spacing (semantic tokens), and Component (component-specific tokens). This hierarchy enables consistent spacing while allowing flexibility for different use cases and easy maintenance.

Semantic Spacing

Semantic spacing tokens provide purpose-based spacing values like $spacing.s, $spacing.m, and $spacing.l. These tokens reference Dimension values and are used for general layout spacing such as margins, padding, and gaps between elements.

Component Spacing

Component spacing tokens are specific to individual components like buttons, cards, and inputs. They reference semantic spacing tokens and provide consistent internal spacing for each component, such as button padding and card content margins.
